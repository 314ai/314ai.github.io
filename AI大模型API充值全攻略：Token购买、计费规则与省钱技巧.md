## 二、Token是什么？如何理解计费单位？

### 2.1 Token的基本概念

在大模型API中，**Token**是计费的基本单位，可以理解为"词块"：

- **英文**：1个Token ≈ 0.75个单词（如"Hello world" ≈ 2 Tokens）
- **中文**：1个汉字 ≈ 1-2个Token（如"你好世界" ≈ 4-6 Tokens）
- **标点符号**：通常占用1个Token

**简单估算：**

- 1000个Token ≈ 750个英文单词 ≈ 500个汉字

### 2.2 302AI的Token计费模式

302AI采用**预付费充值**模式，按实际使用的Token数量扣费：

| 费用类型 | 说明  |
| --- | --- |
| **输入Token（Input）** | 你发送给AI的提示词（Prompt） |
| **输出Token（Output）** | AI返回的回复内容 |
| **总费用** | Input费用 + Output费用 |

**计费示例：**

```
你发送了500 Tokens的提示词 → 输入费用
AI回复了1000 Tokens的内容 → 输出费用
总消耗 = 1500 Tokens
```

## 五、六大省钱技巧

### 技巧1：合理选择模型（省50%+）

**不要所有任务都用GPT-4！**

- 简单任务 → GPT-3.5（成本仅为GPT-4的1/50）
- 复杂任务 → GPT-4-Turbo（成本为GPT-4的1/3）
- 超长文本 → Claude 3 Haiku（128K上下文，价格极低）

### 技巧2：优化Prompt长度（省20-30%）

**减少不必要的Token消耗：**

❌ 低效写法：

```
请你扮演一位专业的Python程序员，帮我解决一个关于数据处理的问题。问题是这样的：我有一个CSV文件...
（已消耗200+ Tokens，还没进入正题）
```

✅ 高效写法：

```
用Python pandas读取CSV并去重：df = pd.read_csv('data.csv')...
（直接给出代码框架，节省Token）
```

**Prompt优化原则：**

- 删除礼貌用语和冗余描述
- 使用英文Prompt（通常比中文节省30% Token）
- 利用系统消息（System Message）设定角色，而非每次重复

### 技巧3：控制输出长度（省10-40%）

通过参数限制AI的回复长度：

```python
# OpenAI API 示例
response = openai.ChatCompletion.create(
    model="gpt-3.5-turbo",
    messages=[{"role": "user", "content": "简述Python特点"}],
    max_tokens=150,  # 限制最多150 Token输出
    temperature=0.7
)
```

### 技巧4：使用缓存和上下文压缩（省30-60%）

**多轮对话优化：**

❌ 每次发送完整历史：

```
User: 你好
Assistant: 你好！有什么可以帮你的？
User: 请继续 [附带以上全部历史]
```

✅ 使用302AI的上下文管理：

```python
# 只发送必要的上下文摘要
conversation_summary = "用户询问Python编程问题，已讨论数据结构"
messages = [
    {"role": "system", "content": "前文摘要：" + conversation_summary},
    {"role": "user", "content": "新问题：..."}
]
```
